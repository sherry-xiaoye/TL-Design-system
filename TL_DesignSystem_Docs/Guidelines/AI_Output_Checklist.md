# AI Output Checklist

生成页面或组件后，必须逐项检查。

## 结构

- 页面是否遵循指定 Pattern？
- 模块顺序是否正确？
- 是否存在不必要的装饰区块？
- 是否存在多层卡片嵌套？

## 组件

- 是否优先使用系统组件？
- 是否重复创建已有基础组件？
- 是否存在 div / span 模拟按钮、输入框、选择器、弹窗等问题？
- 是否把 Empty 作为状态处理，而不是随意自创样式？

## 样式

- 是否存在硬编码颜色？
- 是否存在非 Token 色值？
- 未确认尺寸是否被错误写成 Token？
- 字体、字号、圆角是否符合已确认规范或标记为待定？
- 是否存在超过已确认 Radius Token 的卡片圆角？
- 是否存在大面积渐变、装饰插画、光斑、重阴影？

## 状态

- 是否包含 loading 状态？
- 是否包含 empty 状态？
- 是否包含 error 状态？
- 表单是否包含校验错误状态？
- 高危操作是否包含二次确认？
- 异步提交是否防重复点击？

## 表格

- 表头背景是否使用已确认的表头背景 Token？
- 行分割线是否使用已确认的表格边框 Token？
- 操作列是否固定右侧或保持稳定位置？
- 超长文本是否 ellipsis + Tooltip？
- 是否包含分页？
- 数据量为 0 时是否展示 empty 状态？

## 表单

- 必填项是否有明确标识？
- 错误提示是否展示在输入框下方？
- 底部操作按钮是否右对齐或遵循页面契约？
- 提交时是否有 loading 状态？
