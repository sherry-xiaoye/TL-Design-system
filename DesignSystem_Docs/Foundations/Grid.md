# Grid

本文件定义页面 Body 区域的栅格规则。Grid 不作为颜色、间距、圆角 Token 使用；如需实现，应由前端布局系统映射。

## Body Grid

页面 Body 区域使用 8 栅格，适用于 PC 与 Pad 设备。

| 属性 | 值 | 说明 |
|---|---:|---|
| Columns | `8` | Body 内容区栅格列数 |
| Type | `Stretch` | 栅格随 Body 可用宽度自适应 |
| Margin | `20px` | 与 Body padding 保持一致 |
| Gutter | `20px` | 栅格列间距 |
| 页面最大宽度 | 不设置统一最大宽度 | Body 随屏幕宽度自适应 |
| Breakpoint | 见下方规则 | 用于 PC / Pad 下的布局适配 |

## Desktop Shell

| 区域 | 规格 |
|---|---|
| Header | 高度 `56px` |
| Sidebar-Left | 宽度 `180px` |
| Body | `x=180`, `y=56` |
| Body Padding | `20px` |

## Breakpoint

BMS 不定义移动端优先的完整响应式体系，仅面向 Pad 与 PC 后台使用场景定义通用断点。

| Breakpoint | 范围 | 设备类型 | 布局规则 |
|---|---:|---|---|
| `pad` | `768px - 1199px` | Pad / 小屏横向设备 | 使用 8 栅格，内容随 Body 宽度收缩 |
| `desktop` | `1200px - 1599px` | 常规 PC | 使用 8 栅格，表格与内容区自适应扩展 |
| `wide` | `1600px - 1919px` | 大屏 PC | 使用 8 栅格，优先扩展表格、图表、详情内容 |
| `ultra` | `>= 1920px` | 超宽 PC | 使用 8 栅格，不限制 Body 最大宽度 |

## 适配规则

- PC 与 Pad 均使用 8 栅格。
- 栅格类型为 `Stretch`，列宽随 Body 可用宽度变化。
- 不设置全局统一页面最大宽度。
- 设计稿宽度仅作为设计画布参考，不作为生产环境最大宽度。
- 表格页 Body 可随屏幕宽度扩展。
- 表单页、详情页可按页面模板限制内容阅读宽度，具体规则以 Pattern 为准。
- Dashboard 页可随屏幕扩展，但图表卡片需设置最小可读宽度，具体值待定。
- 超宽屏下不得无限拉伸输入框、按钮、筛选控件。
- 超宽屏下应优先扩展表格列展示、图表区域或详情信息密度。
- Pad 下优先保证信息可读，不强行压缩表格字段。
- 表格字段过多时，使用横向滚动、固定列和列设置，不通过压缩字号解决。

## 栅格使用建议

| 场景 | Pad | Desktop / Wide / Ultra |
|---|---|---|
| 表格区 | 占满 `8` 栅格 | 占满 `8` 栅格 |
| 分页区 | 占满 `8` 栅格 | 占满 `8` 栅格 |
| 筛选项 | 建议每项占 `4` 栅格，每行 2 项 | 建议每项占 `2` 栅格，每行 4 项 |
| 指标卡片 | 建议每项占 `4` 或 `8` 栅格 | 建议每项占 `2` 或 `4` 栅格 |
| 表单内容 | 建议占 `8` 栅格 | 可占 `4`、`6` 或 `8` 栅格，按页面模板确认 |
| 详情分组 | 建议单列展示 | 可单列或双列展示，按页面模板确认 |

## 显隐规则

- 不建议仅因为屏幕变窄隐藏核心业务字段。
- Pad 下可隐藏低频辅助信息，但主键、状态、关键金额、关键操作不可隐藏。
- 列显隐应优先通过 `ColumnSettings` 控制。
- 响应式隐藏规则必须可恢复，不得造成用户无法访问关键操作。

## 禁止事项

- 禁止伪造 Grid Token。
- 禁止在不同页面随意切换布局节奏。
- 禁止为了视觉装饰牺牲表格和表单效率。
- 禁止用固定最大宽度限制表格页 Body 的横向展示能力。
- 禁止通过压缩字号解决 Pad 或窄屏下的字段过多问题。
