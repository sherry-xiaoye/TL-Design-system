# Timeline

## 1.1 用途

用于展示按时间顺序排列的流程、审批、操作记录或状态流转。

## 1.2 视觉规范

- 组件：`Timeline`
- 颜色：优先使用 Semantic Token，其次 Primitive Token，禁止自定义色值
- 尺寸：待定
- 间距：待定
- 圆点 / 连接线：待定
- 字体：待定
- 图标：待定

## 1.3 状态规范

- default：默认时间线展示
- hover：待定
- active / focus：待定
- blur：不适用
- disabled：待定
- error：待定
- loading：待定

## 1.4 交互规则

- 按时间顺序展示节点。
- 节点内容可包含时间、操作人、操作内容、状态说明。
- 是否支持展开收起：待定。
- 异常状态展示规则：待定。

## 1.5 业务场景示例

- 场景 1：【审批业务】中的【审批流程记录】
- 场景 2：【详情业务】中的【操作日志】

## 1.6 前端适配点

- `items` 映射时间线数据
- `time` / `operator` / `content` 字段映射待定
- 组件名建议：`Timeline`

## 1.7 AI 生成约束

- 必须使用系统 `Timeline`。
- 禁止用普通列表模拟时间线。
- 禁止新增未在 Figma 出现的尺寸、颜色、状态、圆角、阴影。
