# TablePage

## 1.1 用途

用于结构化数据查询、筛选、展示、排序、分页、批量操作和行内操作。

适合订单列表、设备列表、用户列表、任务列表、日志列表、财务列表、配置列表等高频业务页面。

## 1.2 页面结构

Body 区域统一采用自上而下的单列布局。

```text
BMS-Header
Sidebar-Left
Body
  Filter-Bar
  Global Actions / List Tools
  Optional Batch Actions
  Table
  Pagination
```

结构规则：

- `Filter-Bar`：筛选条件、查询、重置、展开 / 收起。
- `Global Actions / List Tools`：左侧为全局业务操作，右侧为列表工具类操作。
- `Batch Actions`：进入批量选择后出现，覆盖全局操作区及列表工具区。
- `Table`：承载基础表格、标准业务表格或高级表格。
- `Pagination`：位于表格下方，用于分页数据切换。
- 不得调整为多列页面布局。

## 1.3 视觉规范

- 页面壳：
  - Header：`56px`
  - Sidebar：`180px`
  - Body：`x=180`, `y=56`
  - Body padding：`20px`
  - 主内容宽度：`1220px`
- 颜色：
  - 页面背景优先使用 `Background/Disabled Subtle`
  - 内容容器背景使用 `Background/Default`
  - 表格表头使用 `Background/TableHeader`
  - 表头文本使用 `Text/TableHeader`
  - 表格线使用 `Border/Table`
  - 主文本使用 `Text/Primary`
  - 次级文本使用 `Text/Secondary`
- 字体：
  - 页面 / 区块标题：`Heading/页面内容的标题`
  - 表格正文：`Body/文本`
- 筛选区：
  - 高频条件排列在前。
  - 同一筛选区内控件宽度保持一致；具体宽度以 Figma 或组件文档为准。
  - 默认展示 2 行筛选项，并保证首屏可见。
  - 超出首屏承载时，使用展开 / 收起展示更多筛选。
  - 查询按钮与重置按钮固定在筛选区右侧。
- 表格区：
  - 最小行高：`40`
  - 单元格 padding：`12`
  - 表格行 hover 背景使用 `Background/Zebra`
  - 固定列阴影使用 Figma 已确认 Effect Style；未确认写 `待定`
- 圆角：使用 `Radius/*` Token，未确认写 `待定`
- 间距：使用 `Spacing/*` Token，未确认写 `待定`
- 阴影：使用 Figma 已确认 Effect Style；未确认写 `待定`

## 1.4 组件使用

- 筛选区：
  - 文本输入：`Input`
  - 模糊搜索：`Input` + `Input-Dropdown-Single`
  - 单选少量选项：`Radio`
  - 多选少量选项：`Checkbox`
  - 多选大量选项：`Select.md` 中的 `Select`
  - 日期筛选：`Select.md` 中的 `DatePicker`, `DateTimePicker`
  - 展开 / 收起图标：`Icon` 中的 `Filter-Bar-Unfold`, `Filter-Bar-Fold`
  - 查询 / 重置：`Button`
- 全局操作区：
  - 新建、导入、导出、批量创建入口、页面级设置入口使用 `Button`
  - 危险全局操作使用 `Button-Danger` + `Dialog`
  - 超出展示数量的次级操作使用 `Icon/More` 或 `Icon/MoreFilled` 触发更多菜单
- 列表工具区：
  - 列设置：`ColumnSettings`
  - 排序：`Sort`
  - 开关类工具：`Switch` 或 `Checkbox`
  - 视图切换、显示 / 隐藏字段、紧凑 / 标准视图：具体组件待定
- 批量操作区：
  - 批量选择：`Checkbox`
  - 批量操作按钮：`Button`, `TextButton`, `Button-Danger`
  - 更多批量操作：`Icon/More` 或 `Icon/MoreFilled`
  - 危险批量操作确认：`Dialog`
- 表格区：
  - 表格：`Table-Header`, `Table-Cell`
  - 表头样式：`Default`, `Money`, `Checkbox`, `Empty`, `Action`
  - 单元格状态：`Simple`, `Complex`, `Icon`, `Money`, `Checkbox`, `Action`
  - 行内操作：`TextButton`, `Link`
  - 行内更多菜单：`Icon/More` 或 `Icon/MoreFilled`
  - 排序：`Sort`
  - 列设置：`ColumnSettings`
- 分页区：
  - 分页：`Pagination`

## 1.5 状态规范

- default：默认列表展示状态。
- loading：首次加载、点击查询、重置、分页切换、列表工具生效、批量操作刷新时必须展示。
- empty：无数据时必须展示空状态。
- error：接口失败时展示错误反馈和重试入口。
- selected：选中行后显示批量操作栏，并覆盖全局操作区及列表工具区。
- disabled：无权限操作、不可选行、当前数据态不可执行操作使用禁用状态。
- hover：表格行、按钮、筛选控件按组件 hover 状态展示。
- active / focus：筛选控件和表格操作按组件 active / focus 状态展示。
- blur：输入和选择控件按 Figma `Blur` 状态展示。
- permission denied：如业务存在权限控制，无权限入口需按权限规则隐藏或禁用。

## 1.6 交互规则

### 筛选区

- 筛选区必须包含查询和重置。
- 高频筛选条件排列在前，默认展示 2 行。
- 超出首屏承载时，使用展开 / 收起展示更多筛选。
- 主按钮 `查询` 与次按钮 `重置` 固定在筛选区右侧。
- 当焦点位于可输入筛选项时，支持点击键盘 `Enter` 触发查询。
- 修改筛选条件不会自动刷新表格，用户点击 `查询` 后统一触发刷新。
- 点击 `重置` 后恢复默认筛选值，并刷新列表。
- 日期区间、状态、枚举类条件支持在组件内清空。
- 查询过程中，查询按钮进入 loading 状态；返回结果或修改查询条件后恢复。

### 全局操作区

- 全局操作用于改数据或产生数据，本质是业务动作。
- 适用操作包括：新建、导入、导出、新增订单 / 学生 / 业务对象、批量创建入口、页面级设置入口。
- 左侧放高频主操作，右侧放列表工具类操作。
- 主按钮数量不超过 1 个。
- 其余次级操作按重要性平铺，超出 3 个可收入更多菜单。
- 全局操作对当前列表集合生效，不要求选中行。
- 全局操作与批量操作不得混排；进入批量操作后，批量操作区覆盖全局操作区及列表工具区。

### 列表工具区

- 列表工具不改变数据，只改变展示方式或视角，本质是视图控制 / 页面行为。
- 适用操作包括：列设置、是否只看未达标、视图切换、显示 / 隐藏字段、排序方式、紧凑 / 标准视图。
- 列表工具默认右对齐。
- 数量建议不超过 4 个。
- 开关类功能优先使用 `Switch` 或 `Checkbox`，不得与主操作按钮混淆。
- 列表工具区不允许出现危险操作。
- 主键不可隐藏。
- 冻结列数量上限为 5 列。
- 单字段排序优先在表头触发，多字段排序在工具栏中处理。
- 字段排序在列设置中处理。
- 列表工具对当前列表集合生效，不要求选中行。
- 工具生效后刷新列表。
- 页面再次刷新后保留最近一次配置。

### 批量操作区

- 首列固定为选择列。
- 默认使用多选。
- 支持单选 / 多选，具体业务是否限制单选写 `待定`。
- 支持 `Shift` 连续选择、`Ctrl / Cmd` 补选与反选、`Esc` 清空已选。
- 批量操作区吸附在表格上方。
- 左侧展示高频批量操作和更多菜单。
- 右侧展示选择反馈：`已选择 X 项`、`已选择当前页 X 项` 或 `已跨页选择 X 项`。
- 右侧提供 `全选本页`、`清空选择`。
- 批量操作按钮不超过 3 个，超出收入更多菜单。
- 当已选仅 1 项时，仍显示批量工具栏；按钮按批量能力判断是否可用。
- 点击表头复选框默认选中当前页全部记录，并提示 `已选择当前页 20 项`，其中页条数按当前分页数据确定。
- 批量操作若跨页，必须明确提示 `已跨页选择 X 项`。
- 批量危险操作必须二次确认。
- 批量导出若为异步任务，应给任务反馈，不阻塞页面。

### 表格区

- 表格类型：
  - 基础表格：仅用于查看，不提供复杂交互。适用于日志、只读展示、简单列表。
  - 标准业务表格：提供选择、行操作、更多菜单、批量工具栏。适用于绝大多数后台管理页面。
  - 高级表格：在标准表格基础上，增加行展开、行内编辑、列自定义、复杂冻结、汇总 / 合计、跨页批量操作。适用于预算、财务、课表、产品配置等复杂业务场景。
- 行点击：
  - 字段链接或查看按钮进入详情。
  - 不默认整行跳详情。
- 行内操作：
  - 固定展示，默认展示 1 到 2 个最高频操作，例如查看详情、编辑。
  - 其他操作收纳进更多菜单。
  - 行级操作优先级从左到右为：查看类、编辑类、配置类、风险类。
  - 危险操作必须二次确认。
  - 若操作影响不可逆，弹窗中需写明影响范围。
  - 若操作影响上下游对象，需展示关联提示。
  - 无权限时隐藏操作。
  - 有权限但当前数据态不可执行时置灰，并提供原因提示。
  - 高风险操作不应简单隐藏，应尽量说明为什么不能做。
  - 小计、总计行默认不可编辑，操作列应弱化或隐藏。
- 更多菜单：
  - 低频、破坏性、配置类、带流程确认类动作进入更多操作中。
  - 使用更多图标按钮触发下拉菜单。
  - 点击普通菜单项立即执行。
  - 危险菜单项进入二次确认。
  - 菜单命名使用动词或动词 + 名词，与系统其他页面保持一致。
  - 同类动作不允许同义词混用，例如统一用 `禁用`，不得同页混用 `停用`、`关闭`。
- 行展开与信息分层：
  - 展开内容与主表字段必须有清晰层级关系。
  - 不允许在展开区堆叠过多表单。
  - 若展开内容可独立成为详情页，优先跳详情页。
- 列冻结：
  - 仅当表格存在横向滚动时启用冻结列。
  - 主键固定在左侧。
  - 操作固定在右侧。
  - 状态列、关键金额列、关键决策列可按页面需要冻结。
  - 冻结列必须保证中间内容阅读效率。

### 分页区

- 分页位于表格下方。
- 切换页码后刷新表格数据。
- 每页条数变更后刷新列表，页码重置规则待定。
- 数据量少于或等于 10 条是否隐藏分页，以业务和 Figma 模板为准；未确认写 `待定`。

## 1.7 AI 生成约束

- 必须使用系统 `Table`、`TableHeaderActions`、`Pagination`。
- 必须使用 `Filter-Bar`、全局操作 / 列表工具区、表格区、分页区的自上而下单列结构。
- 必须包含 loading / empty / error / selected / disabled 状态。
- 筛选条件变更不得自动刷新表格，必须通过 `查询` 触发。
- 重置必须恢复默认筛选值并刷新列表。
- 查询按钮在查询过程中必须进入 loading 状态。
- 批量操作出现时，必须覆盖全局操作区及列表工具区。
- 禁止全局操作与批量操作混排。
- 禁止列表工具区出现危险操作。
- 禁止主按钮超过 1 个。
- 禁止行内操作堆叠大按钮。
- 禁止整行默认跳详情。
- 禁止隐藏主键列。
- 禁止冻结列超过 5 列。
- 禁止用 div 拼表格。
- 禁止省略分页，除非业务明确为全量展示或 Figma 模板确认无需分页。
- 禁止新增未在 Figma 或 Token 中确认的颜色、圆角、阴影、状态。
- 未确认的表格尺寸、间距、圆角、阴影必须写 `待定`。
